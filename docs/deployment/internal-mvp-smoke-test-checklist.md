# Internal MVP Smoke Test Checklist

Use this checklist after deploying the local-first Pipeline Defense MVP to Vercel.

- [ ] Open deployed app.
- [ ] Navigate to `/app/pipeline-defense`.
- [ ] Refresh direct route.
- [ ] Create new brief.
- [ ] Edit metadata.
- [ ] Add or import deals.
- [ ] Refresh and confirm local persistence in the same browser.
- [ ] Analyze Deal Risks.
- [ ] Check Review Readiness.
- [ ] Generate This Week's Actions.
- [ ] Enter Review Mode.
- [ ] Export Brief.
- [ ] Print / Save PDF.
- [ ] Confirm no backend/API call was added for Pipeline Defense.
- [ ] Confirm Internal MVP warning banner is visible.
- [ ] Confirm print hides warning/control UI.

## Privacy Check

- [ ] Confirm users understand that Pipeline Defense data is stored in browser `localStorage`.
- [ ] Confirm users understand that clearing browser data may remove saved briefs.
- [ ] Confirm the deployment is protected before internal sharing.
