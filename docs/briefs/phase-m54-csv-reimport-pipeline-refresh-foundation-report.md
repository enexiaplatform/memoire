# Phase M.54 - CSV Re-import & Pipeline Refresh Foundation

## What Was Added

- Added a lightweight CSV refresh mode for `/app/opportunities`.
- Users can now choose between:
  - Import new pipeline
  - Refresh existing pipeline
- Refresh mode compares incoming CSV rows against existing opportunities using `accountName + opportunityName`.
- Refresh preview classifies rows as:
  - New opportunity
  - Existing changed
  - Existing unchanged
  - Possible duplicate
  - Invalid row
- Changed rows show field-level diffs with current value, imported value, and selectable update checkboxes.
- Added local import/refresh batch history using `memoire.importBatches.v1`.
- Added read-only positioning copy: Memoire updates a private working copy and never writes back to CRM.

## Refresh Comparison Logic

Primary matching uses:

- `accountName`
- `opportunityName`

Possible duplicate detection also checks:

- account partial match
- opportunity/product token overlap
- matching estimated value
- matching expected close period

The refresh preview is generated by:

- `compareImportedOpportunities()`
- `detectOpportunityChanges()`
- `preparePipelineRefreshPreview()`

## Field Update Rules

Default selected refresh fields are safe CRM/source fields:

- stage
- estimatedValue
- currency
- expectedClosePeriod
- productOrSolution
- nextAction

Protected/enriched fields are shown but not default-overwritten unless selected:

- evidence
- missingContext
- forecastEvidenceCategory
- status

Memoire does not overwrite stakeholders, objections, action outcomes, playbook patterns, assets, or Pipeline Defense content during refresh.

## User-Enriched Data Protection

Refresh does not silently replace user enrichment. The user must preview changes and explicitly apply them.

Existing enriched records remain intact:

- decision maker
- budget owner
- procurement path
- technical criteria
- objection debt
- stakeholder map
- objection ledger
- action outcomes
- playbook/assets links

## Import Batch History

Local-only batch records are stored in:

`memoire.importBatches.v1`

Each batch records:

- mode
- createdAt
- fileName
- rowCount
- newCount
- changedCount
- skippedCount
- invalidCount

No Supabase SQL is required.

## Manual QA Checklist

1. Open `/app/opportunities`.
2. Click `Import CSV`.
3. Import a sample CSV as new opportunities.
4. Re-import the same CSV in refresh mode.
5. Confirm unchanged/skipped behavior.
6. Modify stage, value, close period, or next action in the CSV.
7. Refresh again and confirm changed opportunities preview renders.
8. Apply refresh and confirm selected base fields update.
9. Confirm enriched fields are not overwritten silently.
10. Confirm import/refresh history renders.
11. Confirm enrichment signals still render.
12. Generate Pipeline Defense Brief from refreshed opportunities.
13. Confirm add/edit/delete opportunity still works.
14. Confirm previous CSV import still works.
15. Confirm demo sandbox still works.
16. Run `npm run build`.
17. Run `npm run lint`.

## Known Limitations

- This is not live Salesforce/HubSpot sync.
- Matching is rule-based and intentionally conservative.
- Refresh metadata is local-only; cloud schema is unchanged.
- Field-level update selection is designed for pipeline source fields, not full CRM data reconciliation.
